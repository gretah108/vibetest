# 개인 홍보용 원페이지 웹사이트 제작 스펙

## 1. 목표

개인을 홍보하기 위한 세련된 원페이지 포트폴리오 사이트를 만든다.
사용자는 코딩을 못하므로, 코드 구조는 단순하고 수정하기 쉽게 작성한다.

전체 분위기는 **애플 공식 홈페이지처럼 미니멀하고 고급스럽게**, 넓은 여백, 부드러운 애니메이션, 큰 타이포그래피를 사용한다.

## 2. 전체 콘셉트

* 스타일: Apple-like, minimal, premium, clean
* 메인 컬러: 회색 계열
* 배경: 밝은 회색 또는 거의 흰색
* 포인트 컬러: 진한 회색, 검정, 실버 계열
* 레이아웃: 한 페이지 스크롤형
* 메뉴 클릭 시 해당 섹션으로 부드럽게 이동
* 반응형 지원: PC, 태블릿, 모바일

## 3. 메뉴 구조

상단 고정 네비게이션 메뉴를 만든다.

메뉴:

* 홈
* 자기소개
* 포트폴리오
* 취미

메뉴 수가 적으므로 별도 페이지를 만들지 않고, 한 페이지 안에서 섹션별로 길게 이어지는 구조로 만든다.

## 4. 페이지 섹션 구성

### 4.1 홈 섹션

첫 화면에 크게 보이는 히어로 영역을 만든다.

포함 내용:

* 큰 제목: “안녕하세요, 저는 [이름]입니다.”
* 서브 문구: “나를 소개하고, 내가 만든 것들과 좋아하는 것들을 담은 공간입니다.”
* 부드러운 회색 그라데이션 배경
* Apple 홈페이지처럼 큰 글자, 넓은 여백, 중앙 정렬
* 스크롤을 유도하는 작은 애니메이션 아이콘

디자인 요구:

* 화면 높이 100vh
* 텍스트가 천천히 나타나는 fade-in 애니메이션
* 배경에 은은한 blur 또는 glassmorphism 효과 사용 가능

### 4.2 자기소개 섹션

내 이력과 사진을 보여주는 영역이다.

포함 내용:

* 프로필 사진 영역
* 이름
* 짧은 자기소개 문장
* 이력 타임라인

예시 구조:

* 왼쪽: 내 사진
* 오른쪽: 자기소개 텍스트와 이력

이력은 카드 또는 타임라인 형태로 보여준다.

예시 항목:

* [연도] [이력 내용]
* [연도] [이력 내용]
* [연도] [이력 내용]

사진은 사용자가 나중에 쉽게 교체할 수 있도록 `profile.jpg` 같은 파일명으로 연결한다.

### 4.3 포트폴리오 섹션

포트폴리오는 총 4개를 보여준다.

각 포트폴리오 카드는 다음 내용을 포함한다:

* 프로젝트 제목
* 한 줄 설명
* 사용 기술 또는 역할
* 자세히 보기 버튼

포트폴리오 카드 개수:

1. 포트폴리오 1
2. 포트폴리오 2
3. 포트폴리오 3
4. 포트폴리오 4

디자인 요구:

* PC에서는 2x2 카드 그리드
* 모바일에서는 1열 카드
* 카드에 마우스를 올리면 살짝 떠오르는 hover 효과
* 카드 배경은 흰색 또는 밝은 회색
* 그림자와 둥근 모서리 사용

### 4.4 취미 섹션

취미는 총 3개를 보여준다.

각 취미 카드는 다음 내용을 포함한다:

* 취미 이름
* 짧은 설명
* 아이콘 또는 이미지

취미 카드 개수:

1. 취미 1
2. 취미 2
3. 취미 3

디자인 요구:

* 3개의 카드가 가로로 배치
* 모바일에서는 세로 배치
* 부드러운 등장 애니메이션 적용
* 감성적인 문구와 여백을 사용

## 5. 디자인 상세

### 컬러 팔레트

다음 회색 계열을 사용한다.

* 배경색: `#F5F5F7`
* 기본 텍스트: `#1D1D1F`
* 보조 텍스트: `#6E6E73`
* 카드 배경: `#FFFFFF`
* 선 색상: `#D2D2D7`
* 진한 포인트: `#333333`

### 폰트

* 기본 폰트: `Pretendard`, `Noto Sans KR`, `Apple SD Gothic Neo`, sans-serif
* 제목은 굵고 크게
* 본문은 읽기 쉽게 적당한 행간 사용

### 애니메이션

Apple 홈페이지 느낌을 위해 다음 효과를 넣는다.

* 섹션 진입 시 fade-up 애니메이션
* 메뉴 클릭 시 smooth scroll
* 카드 hover 시 transform 효과
* 히어로 텍스트 fade-in
* 배경에 은은한 blur 효과

단, 과하지 않게 미니멀하게 적용한다.

## 6. 기술 요구사항

가능하면 가장 단순한 구조로 만든다.

필수:

* HTML
* CSS
* JavaScript

또는 React로 만들어도 된다면:

* React
* CSS Modules 또는 일반 CSS

코딩을 모르는 사용자가 수정하기 쉽도록, 텍스트 데이터는 한 곳에서 관리할 수 있게 작성한다.

예:

```js
const portfolioItems = [
  {
    title: "포트폴리오 1",
    description: "프로젝트 설명을 입력하세요.",
    role: "역할 또는 사용 기술"
  }
];
```

## 8. 반응형 요구사항

### PC

* 넓은 여백
* 중앙 정렬
* 포트폴리오 2x2
* 취미 3열

### 모바일

* 메뉴는 작게 정리하거나 햄버거 메뉴 사용
* 모든 카드 1열 배치
* 텍스트 크기 자동 축소
* 사진과 자기소개는 세로 배치

## 9. UX 요구사항

* 사용자가 스크롤하면서 자연스럽게 나를 알아갈 수 있어야 한다.
* 메뉴를 누르면 해당 섹션으로 부드럽게 이동해야 한다.
* 첫 화면에서 고급스럽고 전문적인 인상을 줘야 한다.
* 불필요한 장식은 줄이고, 여백과 타이포그래피로 세련되게 표현한다.
* 코딩을 모르는 사람이 텍스트와 이미지만 쉽게 교체할 수 있어야 한다.

### 이름

### 자기소개

* `2024 - 주요 활동 또는 경력 입력`
